- POST /documents/split  
  - Input: multipart/form-data: `pdf` (file, required), optional fields `category` (string), `company` (string), `department` (string), `competence_period` (string)
  - Restituisce: `{ status: "queued", message: string, job_id: string, uploaded_document_id: integer }`  
  - WebSocket (sottoscrivere `job_id`):  
    - `split_completed`: `{ status: "success" | "error", original_pages: int, mini_pdfs_count: int, mini_pdfs: [string], message?: string }` inviato al termine del processo di split.  
    - `document_processed`: `{ event: "document_processed", status: "success" | "error", extracted_document_id: int, filename?: string, ocr_text?: string, extracted_names?: [string], extracted_document_data?: { company: string|null, department: string|null, date: string|null, type: string|null, reason: string|null, competence: string|null }, extracted_confidence?: { recipient: float, date: float, company: float, department: float, type: float, reason: float, competence: float }, matched_recipient?: { id: int, name: string, email: string, employee_code: string }, fallback_text?: string, message?: string }` inviato quando un mini‑PDF è stato processato (successo o errore).  
    - `processing_completed`: `{ status: "success" | "error" }` inviato quando il run è finito.

- POST /documents/test_data  NON CREDO LA USERETE
  - Input: multipart/form-data: `pdf` (file, required)
  - Restituisce: `{ status: "queued", message: string, job_id: string }`  
  - WebSocket: può ricevere `document_queued` e poi `document_processed`/`document_error` per il singolo item.

- GET /documents/uploads/:uploaded_document_id/extracted  
  - Input: path param `uploaded_document_id` (integer)
  - Restituisce:  
    {
      uploaded_document: { id, original_filename, page_count, created_at, ... },
      extracted_documents: [ extracted_document_payload, ... ]
    }  
  - `extracted_document_payload` include: `id, sequence, status, page_start, page_end, metadata, recipients, confidence, recipient_name, matched_employee, error_message, processed_at, pdf_download_url, process_time_seconds, created_at, updated_at`  
  - WebSocket: non strettamente necessario (è snapshot), ma il canale invierà aggiornamenti `document_processed` / `document_error` quando i singoli documenti vengono processati.

- GET /documents/extracted/:id  
  - Input: path param `id` (integer)
  - Restituisce: `{ extracted_document: extracted_document_payload }` (vedi sopra)  
  - WebSocket: aspettarsi eventuali `document_processed` aggiornamenti per lo stesso `id`.

- GET /documents/extracted/:id/pdf  
  - Input: path param `id` (integer)  
  - Restituisce: stream PDF (attachment) generato on‑demand.  
  - WebSocket: nessun messaggio specifico; UI scarica quando l'utente clicca.

- PATCH /documents/extracted/:id/reassign_range  
  - Input: path param `id` (integer) and body either JSON `{ "page_start": int, "page_end": int }` or form-data `page_range` (string "X-Y")  
  - Restituisce: `{ status: "queued", message: string, extracted_document_id, page_start, page_end }`  
  - WebSocket: nessun evento aggiuntivo specifico viene inviato; ascolta `document_processed` / `processing_completed` per sapere quando quel documento è stato rielaborato.

- GET /processing_runs/:job_id (consigliato)  
  - Input: path param `job_id` (string)  
  - Restituisce: `{ job_id, status, total_documents, processed_documents, started_at, completed_at, items: [{sequence, status, extracted_document_id}] }`  
  - Uso: utile per polling periodico se WebSocket non disponibile.

WebSocket: come sottoscrivere e cosa aspettarsi
- Canale: `DocumentProcessingChannel` (ActionCable)  
- Subscribe identifier (client): `{"channel":"DocumentProcessingChannel","job_id":"<JOB_ID>"}`  
- Messaggi che il client può ricevere (tipo + payload):
  - `split_completed`: `{ status: "success" | "error", original_pages: int, mini_pdfs_count: int, mini_pdfs: [string], message?: string }` (fine split)
  - `document_processed`: `{ event: "document_processed", status: "success" | "error", extracted_document_id: int, filename?: string, ocr_text?: string, extracted_names?: [string], extracted_document_data?: {...}, extracted_confidence?: float, matched_recipient?: {id,name,email,employee_code}, fallback_text?: string, message?: string }` (fine elaborazione singolo item)
  - `processing_completed`: `{ status: "success" | "error" }` (fine run)
